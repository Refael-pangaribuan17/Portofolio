The IoT-enabled gas and fire detection system can also be integrated with other smart home or industrial automation systems, providing a comprehensive approach to safety management. By leveraging IoT technology, this system ensures real-time monitoring, enhances safety measures, and provides peace of mind by allowing for immediate intervention in case of emergencies.

This innovative trash can features a dual-compartment system where the metal waste and non-metal waste are sorted automatically or manually into separate bins, making recycling more efficient and organized. SolidWorks allows for precise modeling of these components, ensuring that the trash can is both functional and easy to manufacture. The trash can may also incorporate features such as a lid with an automatic opening mechanism for hands-free disposal, sensors to detect when the bin is full, and a foot pedal for easier operation. Additionally, the design can include considerations for ergonomics, durability, and ease of maintenance, ensuring that the product is suitable for both residential and commercial use. SolidWorks offers advanced simulation tools that can test the design for structural integrity and functionality, ensuring that the sorting bins are reliable, safe, and effective for long-term use. The final product not only simplifies waste disposal but also supports sustainable practices by encouraging proper recycling of materials.

Simple Game

Magic Square C Language

A Magic Square game for two players, developed in C language, is an interactive and competitive game where each player takes turns filling a 3x3 grid with numbers from 1 to 9, aiming to create a magic square. In a 3x3 magic square, the sum of each row, column, and diagonal must equal 15. The game begins with an empty grid, and players alternate selecting a number to place in an empty spot. The game ensures that each number is unique and between 1 and 9, providing input validation to prevent players from choosing a number that has already been placed or selecting an occupied spot. Players are prompted to enter the number they wish to place and the grid position, and the game checks if the move is valid. The game continues until one player successfully arranges the numbers in such a way that every row, column, and diagonal sums to 15, thereby completing the magic square. At that point, the game ends, and the winner is declared. If all spots are filled without a winner, the game results in a draw. The game logic ensures smooth gameplay by managing turn-taking, checking for winning conditions, and displaying the updated grid after each move, providing an engaging challenge for both players.

Kubernetes was then utilized for container orchestration, allowing the deployment, scaling, and management of containers across multiple nodes in the simulated environment. With Minikube, a lightweight Kubernetes implementation, I was able to create a local development environment that mimicked real-world Kubernetes clusters. This setup included defining Kubernetes manifests for pods, services, and deployments to ensure proper resource allocation, load balancing, and fault tolerance. The project also focused on the seamless integration of these tools to achieve a fully automated and orchestrated pipeline. For example, Ansible playbooks were used to provision Kubernetes clusters on Minikube, while Docker images were directly integrated into Kubernetes deployments for containerized workloads. The orchestration setup ensured high availability, scalability, and ease of management, replicating real-world deployment scenarios. By completing this project, I gained hands-on experience in automating infrastructure management, containerization, and orchestration. It highlighted the importance of combining tools like Ansible, Docker, Kubernetes, and Minikube to build efficient, reliable, and scalable systems that can adapt to dynamic application requirements in both simulated and real-world environments.

Simple Game

Tic Tac Toe

A Tic Tac Toe game developed in C language is a simple but engaging implementation of the classic two-player game where players take turns placing "X" and "O" on a 3x3 grid. The goal is to be the first to get three marks in a row, either horizontally, vertically, or diagonally. The game starts with an empty grid, which is represented as a 2D array in C. Each player alternates between Player 1 (who plays "X") and Player 2 (who plays "O"), choosing a number between 1 and 9 corresponding to an empty spot on the grid. The program checks if the chosen spot is available and prompts the player to choose again if the spot is already taken. The game includes input validation to ensure that players enter valid numbers within the range and only choose unoccupied positions. After each move, the program updates the grid and checks whether the current player has won by achieving three consecutive marks in a row, column, or diagonal. If a player achieves this, they win the game, and the program announces the winner. If all spots on the grid are filled and no player has won, the game ends in a draw. The program runs in a loop, continuously alternating between players and checking for winning or draw conditions until the game is over. This implementation not only provides a fun, interactive experience but also reinforces concepts such as arrays, loops, conditional statements, and user input handling in C programming.
